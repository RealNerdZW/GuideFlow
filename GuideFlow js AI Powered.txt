GuideFlow.js — AI-Powered Product Tour Library

Context & Mission
Build GuideFlow.js, a next-generation JavaScript library for product tours, contextual highlights, and in-app guidance. The three leading libraries in this space are Driver.js (DOM overlay + popover highlighting), Intro.js (attribute-driven step tours), and OnboardJS (headless React-first flow engine). After deep analysis, all three share critical gaps this library must solve.

Competitive Gap Analysis (Build Beyond These)
-Driver.js gaps: No persistence, no analytics, no AI, no framework-native adapters, manual step configuration only, no conditional branching, no multi-user/role segmentation, no no-code editor.
-Intro.js gaps: Attribute-based approach is fragile at scale, no AI, no headless mode, limited theming, no branching logic, poor TypeScript support, no analytics.
-OnboardJS gaps: React-only, no DOM highlighting/spotlight capability, no AI, no built-in visual components, requires heavy DIY UI work, no DevTools.
Universal gaps across all three: No AI-powered features, no visual flow builder, no A/B testing support, no real-time analytics, no ambient/persistent hint mode, no natural language tour creation, no behavioral trigger system, no cross-session progress, no team collaboration tooling.

Core Architecture Requirements
Package structure:
@guideflow/core         — framework-agnostic engine (zero dependencies)
@guideflow/react        — React hooks + components
@guideflow/vue          — Vue 3 composables + components
@guideflow/svelte       — Svelte stores + components
@guideflow/ai           — AI brain module (provider-agnostic)
@guideflow/analytics    — telemetry + heatmap module
@guideflow/devtools     — browser extension + visual studio

The core must be tree-shakeable, under 12kb gzipped (base), and run in SSR environments without DOM access errors.

Feature Requirements by Category
1. Tour & Spotlight Engine (Foundation)
Implement all capabilities of Driver.js and Intro.js, then go further:

Smooth animated spotlight cutout over any DOM element (CSS clip-path + SVG mask approach)
Popover with smart auto-positioning (12 placements, viewport collision detection, scroll-into-view)
Multi-step tours with linear, branching, and looping step graphs — not just arrays
Step-level conditions: showIf: (context) => boolean — skip steps based on user role, feature flags, previous actions
Async step resolution: each step can await data before rendering
Keyboard navigation (arrow keys, Escape, Tab trapping inside popover)
Click-through mode: allow user to interact with highlighted element while tour is active
Virtual/floating steps with no target element (modal-style)
Hotspot mode: persistent pulsing beacon on elements with tooltip on hover — no modal, no flow blocking
Hint system: ambient markers (like Intro.js hints) that stay on screen persistently

2. AI Brain Module (@guideflow/ai)
This is the primary differentiator. No existing library has any AI capability.
Natural language tour creation:
const tour = await guideflow.ai.generate({
  prompt: "Create an onboarding tour for a project management dashboard, 
           focusing on creating the first task and inviting a team member",
  dom: document, // library introspects the live DOM
  provider: "anthropic", // or "openai", "ollama"
})

The AI inspects the current DOM, identifies relevant interactive elements, and generates a complete step array with titles, descriptions, and target selectors.
Behavioral intent detection: Monitor user behavior passively (mouse movement, click patterns, time-on-element, rage clicks) and surface contextual help automatically:
guideflow.ai.watch({
  onConfused: (element, context) => guideflow.hint(element, { auto: true }),
  onStuck: (element, duration) => guideflow.suggest(element),
})

Adaptive tour compression: Track which steps users consistently skip and automatically shorten the tour for future users matching the same profile.
Conversational help panel: Optionally mount a chat widget powered by the AI module where users can ask "how do I do X?" and get a real-time guided answer — the library highlights relevant elements on the page as it explains.
Provider interface:
interface AIProvider {
  generateSteps(context: DOMContext, prompt: string): Promise<Step[]>
  detectIntent(events: UserEvent[]): Promise<IntentSignal>
  answerQuestion(question: string, context: PageContext): Promise<GuidedAnswer>
}

Support for OpenAI, Anthropic, local Ollama, and a mock provider for testing.
3. State Machine & Flow Engine
Replace the simple "array of steps" model with a proper state machine:
const flow = guideflow.createFlow({
  id: "onboarding",
  initial: "welcome",
  states: {
    welcome: { on: { NEXT: "role_select" } },
    role_select: {
      on: {
        CHOOSE_DEVELOPER: "dev_tour",
        CHOOSE_DESIGNER: "design_tour",
        CHOOSE_MANAGER: "mgr_tour",
      }
    },
    dev_tour: { steps: devSteps, on: { DONE: "completion" } },
    // ...
  }
})

Supports XState-compatible format as an optional adapter. Persist state to localStorage, IndexedDB, or a custom backend. Resume interrupted tours from exact step on page reload.
4. Renderer & Theming
Headless mode (like OnboardJS) — provide only logic, let developers render their own UI via render props / slots / composables.
Themed mode — a beautiful default UI built on CSS custom properties:
--gf-popover-bg
--gf-popover-text
--gf-accent-color
--gf-overlay-opacity
--gf-border-radius
--gf-shadow
--gf-font-family

Support dark mode, RTL, and high-contrast accessibility modes out of the box. Ship 5 built-in themes (minimal, bold, glass, brutalist, enterprise). Allow design token import from Tailwind, Radix, or shadcn/ui token formats.
5. Analytics & Telemetry (@guideflow/analytics)
No existing library has this built in.

Track: step views, completion rate per step, abandonment points, time-per-step, skip events, click events on highlighted elements
Export to: PostHog, Mixpanel, Amplitude, Segment, or custom webhook
Built-in dashboard component (mountable React/Vue component) showing funnel visualization
A/B testing API:
guideflow.experiment({
  id: "tour-length-test",
  variants: {
    control: fullTourSteps,
    short: shortTourSteps,
  },
  split: 0.5,
  metric: "completion_rate",
})

6. Persistence & Multi-Session Progress
jsguideflow.configure({
  persistence: {
    driver: "localStorage", // or "indexedDB", "custom"
    key: (userId) => `gf:${userId}:progress`,
    ttl: 30 * 24 * 60 * 60 * 1000, // 30 days
  }
})

Track per-user progress, "don't show again" preferences, partially completed tours, and step-level acknowledgements. Sync across tabs via BroadcastChannel.
7. Developer Experience

Full TypeScript with strict generics — step payloads typed, context typed, no any
Vite + Rollup build with ESM, CJS, and IIFE outputs
Zero mandatory CSS import — styles injected dynamically or via explicit import
Framework-specific DX:
tsx
// React
const { start, next, prev, state } = useTour("onboarding")
<TourStep id="step-1" target="#dashboard-header">
  <p>This is your dashboard</p>
</TourStep>

// Vue
const tour = useTour("onboarding")
<GuideStep target="#dashboard-header" v-slot="{ next }">...</GuideStep>

Data-attribute API for simple cases (Intro.js compat mode):

html<button data-gf-step="1" data-gf-title="Save changes" data-gf-body="Click here to save">

8. DevTools & Visual Studio (@guideflow/devtools)
A browser extension + visual drag-and-drop editor:

Point-and-click step builder: click any element on the page to add it as a tour step
Live preview of tours without deploying code
Step reordering via drag-and-drop
AI assist inside the editor: "Generate a tour for this page" button
Flow graph visualizer showing branching logic
Export to JSON, TypeScript, or directly to your codebase via a CLI push command
Team collaboration: share flows via URL, leave comments on steps, version history

9. Accessibility & Internationalisation

WCAG 2.2 AA compliant out of the box
Full ARIA: role="dialog", aria-labelledby, aria-describedby, focus management, focus trap in popovers
Screen reader announcements for step transitions
RTL support (popover flips, progress direction)
i18n API:
js
guideflow.i18n.register("fr", { next: "Suivant", prev: "Précédent", close: "Fermer" })
guideflow.i18n.use("fr")

Non-Functional Requirements

Zero runtime dependencies in @guideflow/core
Works with React 17+, Vue 3+, Svelte 4+, and any vanilla JS project
SSR safe — no window/document access at module load time; defer to runtime
Supports Shadow DOM hosts (useful for web components)
Content Security Policy compliant — no eval, no inline styles injected without nonce support
Works in iframes (modal-based apps, embedded tools)
Bundle size targets: core ≤ 12kb, full with AI ≤ 28kb (excluding AI SDK peer deps)
Tested: Vitest unit tests, Playwright E2E, Axe accessibility tests in CI
