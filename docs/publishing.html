Publishing to npm & jsDelivr

This guide covers everything you need to publish GuideFlow packages to npm and serve them via jsDelivr's global CDN. It is written for the GuideFlow monorepo which uses pnpm workspaces, Turborepo, and Changesets.

1. Overview & Architecture

GuideFlow ships as a set of scoped npm packages under the @guideflow organisation:

PackageDescription
@guideflow/coreFramework-agnostic engine (zero deps)
@guideflow/reactReact bindings & hooks
@guideflow/vueVue 3 plugin & composables
@guideflow/svelteSvelte integration
@guideflow/aiAI-powered guide brain
@guideflow/analyticsAnalytics collector & transports
@guideflow/cliCLI tooling & studio
@guideflow/devtoolsBrowser DevTools extension

Internal-only packages (@guideflow/docs, @guideflow/e2e, @guideflow/storybook) are excluded from publishing via the Changesets ignore list.

2. Prerequisites

  • Node.js ≥ 18.0.0
  • pnpm ≥ 8.0.0 (corepack enable to activate)
  • Git installed and configured
  • An npm account (free or paid) at npmjs.com/signup
  • Access to the @guideflow npm organisation (ask a maintainer to invite you)
# Verify your local environment
node --version    # must be ≥ 18
pnpm --version    # must be ≥ 8
git --version

3. Setting Up Your npm Account

3.1 Create an Account

  1. Go to https://www.npmjs.com/signup
  2. Fill in username, email, and password.
  3. Verify your email address.

3.2 Enable Two-Factor Authentication (Recommended)

  1. Go to Account → Security on npmjs.com.
  2. Enable 2FA for authorization and publishing.
  3. Scan the QR code with an authenticator app (Google Authenticator, Authy, 1Password, etc.).
Note: If you enable 2FA for publishing, you will need to supply an OTP code when publishing from the command line. CI/CD tokens can bypass this — see Section 10.

3.3 Create / Join the @guideflow Organisation

If the org does not exist yet:

  1. Go to npmjs.com/org/create.
  2. Enter guideflow as the org name.
  3. Choose the free/open-source plan (unlimited public packages).

If the org already exists: ask a current owner to invite your npm username under Organisation → Members → Invite.

4. Authenticating Locally

# Log in to the npm registry
npm login

# Verify the logged-in user
npm whoami

# Confirm org membership
npm org ls guideflow

This writes a token to ~/.npmrc. Never commit this file to version control.

5. Creating a Changeset

GuideFlow uses Changesets to manage versioning and changelogs. Every publishable change must have a changeset.

5.1 Add a Changeset

# From the monorepo root:
pnpm changeset

You will be prompted to:

  1. Select packages that changed (use Space to toggle, Enter to confirm).
  2. Choose bump type for each: patch, minor, or major.
  3. Write a summary of the change.

This creates a markdown file in .changeset/:

# Example: .changeset/happy-tiger-42.md
---
"@guideflow/core": minor
"@guideflow/react": patch
---

Added spotlight animation options and updated React hook signatures.

5.2 Commit the Changeset

git add .changeset/
git commit -m "chore: add changeset for spotlight animation"

6. Versioning Packages

When you are ready to cut a release, consume all pending changesets:

# Bump versions and update CHANGELOGs
pnpm version-packages

This runs changeset version under the hood and will:

  • Update version in each affected package.json.
  • Update internal workspace dependency ranges (the updateInternalDependencies: "patch" policy).
  • Append entries to each package's CHANGELOG.md.
  • Delete consumed changeset files from .changeset/.
# Review the changes, then commit
git add -A
git commit -m "chore: version packages"
git push

7. Publishing to npm

7.1 One-Command Publish

# Build everything and publish changed packages
pnpm publish-packages

This single command runs the root's publish-packages script, which:

  1. Builds all publishable packages via Turborepo (turbo run build, excluding docs/e2e/storybook).
  2. Runs changeset publish to publish every package with a bumped version to the npm registry.
  3. Creates git tags like @guideflow/core@0.2.0 for each published version.

7.2 Step-by-Step (Manual)

If you prefer fine-grained control:

# 1. Build all packages
pnpm build

# 2. Publish a single package (from its directory)
cd packages/core
npm publish --access public

# 3. Or publish all changed packages
cd ../..   # back to root
pnpm changeset publish

7.3 With 2FA / OTP

# Supply an OTP from your authenticator app
npm publish --access public --otp 123456
Important: Each @guideflow/* package must be published with --access public the first time. Subsequent publishes inherit this setting. The Changesets config already sets "access": "public", so changeset publish handles this automatically.

7.4 Push Git Tags

# Push the version commit and tags to remote
git push --follow-tags

8. Scoped Package Access (@guideflow/*)

By default, scoped packages on npm are private. GuideFlow packages are open-source and must be published as public.

MethodHow
Changesets config (automatic) Already set: "access": "public" in .changeset/config.json
CLI flag npm publish --access public
Per-package package.json Add "publishConfig": { "access": "public" }

9. Using jsDelivr CDN

jsDelivr is a free, public CDN that automatically mirrors every package on npm. Once a package is published to npm, it is available on jsDelivr within minutes — no extra configuration required.

9.1 URL Format

https://cdn.jsdelivr.net/npm/<package>@<version>/<file>

9.2 GuideFlow CDN URLs

AssetURL
Core ESM (latest) https://cdn.jsdelivr.net/npm/@guideflow/core/dist/index.js
Core ESM (pinned) https://cdn.jsdelivr.net/npm/@guideflow/core@0.1.0/dist/index.js
Core CJS https://cdn.jsdelivr.net/npm/@guideflow/core@0.1.0/dist/index.cjs
Core CSS https://cdn.jsdelivr.net/npm/@guideflow/core@0.1.0/dist/styles/index.css
React bindings https://cdn.jsdelivr.net/npm/@guideflow/react@0.1.0/dist/index.js
AI package https://cdn.jsdelivr.net/npm/@guideflow/ai@0.1.0/dist/index.js

9.3 Version Ranges on jsDelivr

SpecifierExampleBehaviour
Exact @guideflow/core@0.2.1 Always this exact version (recommended for production)
Semver range @guideflow/core@^0.2.0 Latest compatible version (auto-updates patches & minors)
Latest @guideflow/core (no version) Always the latest published version
Dist tag @guideflow/core@next Follows the next npm dist-tag

9.4 Using GuideFlow via CDN in HTML

<!-- Load the stylesheet -->
<link rel="stylesheet"
      href="https://cdn.jsdelivr.net/npm/@guideflow/core@0.1.0/dist/styles/index.css">

<!-- Load the ES module -->
<script type="module">
  import { createGuideFlow } from
    'https://cdn.jsdelivr.net/npm/@guideflow/core@0.1.0/dist/index.js';

  const tour = createGuideFlow({
    steps: [
      { target: '#welcome', title: 'Welcome!', content: 'Let\u2019s get started.' },
      { target: '#feature', title: 'Key Feature', content: 'Check this out.' }
    ]
  });
  tour.start();
</script>

9.5 Using the jsDelivr “Combine” Feature

Load multiple files in a single HTTP request:

<script type="module"
  src="https://cdn.jsdelivr.net/combine/npm/@guideflow/core@0.1.0/dist/index.js,npm/@guideflow/ai@0.1.0/dist/index.js">
</script>

9.6 Minified Versions

Append .min before the extension for on-the-fly minification:

https://cdn.jsdelivr.net/npm/@guideflow/core@0.1.0/dist/index.min.js
https://cdn.jsdelivr.net/npm/@guideflow/core@0.1.0/dist/styles/index.min.css

9.7 Browsing Package Files

View all files in a published package:

https://cdn.jsdelivr.net/npm/@guideflow/core@0.1.0/

9.8 Purging the jsDelivr Cache

jsDelivr caches files for up to 7 days. To force-refresh after a new publish:

curl -X POST "https://purge.jsdelivr.net/npm/@guideflow/core@0.1.0/dist/index.js"

Or use their Purge Tool.

10. CI/CD Automation (GitHub Actions)

Automate the entire publish pipeline with GitHub Actions.

10.1 Add the npm Token to GitHub Secrets

  1. On npmjs.com, go to Access Tokens → Generate New Token.
  2. Choose Automation (bypasses 2FA, intended for CI).
  3. Copy the token.
  4. In your GitHub repo, go to Settings → Secrets → Actions → New repository secret.
  5. Name: NPM_TOKEN, Value: paste token.

10.2 Workflow File

Create .github/workflows/publish.yml:

name: Publish Packages

on:
  push:
    branches: [main, master]

concurrency:
  group: ${{ github.workflow }}-${{ github.ref }}
  cancel-in-progress: true

jobs:
  publish:
    runs-on: ubuntu-latest
    permissions:
      contents: write      # push tags
      id-token:  write     # npm provenance
    steps:
      - name: Checkout
        uses: actions/checkout@v4
        with:
          fetch-depth: 0   # full history for changesets

      - name: Setup pnpm
        uses: pnpm/action-setup@v2
        with:
          version: 8

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: pnpm
          registry-url: https://registry.npmjs.org

      - name: Install dependencies
        run: pnpm install --frozen-lockfile

      - name: Build all packages
        run: pnpm build

      - name: Run tests
        run: pnpm test

      - name: Create Release Pull Request or Publish
        uses: changesets/action@v1
        with:
          publish: pnpm publish-packages
          version: pnpm version-packages
          title: "chore: version packages"
          commit: "chore: version packages"
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          NPM_TOKEN: ${{ secrets.NPM_TOKEN }}
          NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }}

10.3 How the Workflow Works

  1. No pending changesets: The action runs changeset publish, publishing any packages whose version was bumped but not yet published.
  2. Pending changesets exist: The action opens (or updates) a Release PR that runs changeset version. Merging this PR triggers a re-run that publishes.
Result: Once merged, every @guideflow/* package with a version bump is automatically published to npm and immediately available on jsDelivr.

11.1 Check npm

12. Rolling Back a Bad Release

12.1 Deprecate a Version

# Mark a version as deprecated (users see a warning on install)
npm deprecate @guideflow/core@0.2.0 "Known bug in spotlight — use 0.2.1 instead"

12.2 Unpublish (within 72 hours)

# npm allows unpublishing within 72 hours
npm unpublish @guideflow/core@0.2.0
Warning: Unpublishing can break other people's builds. Prefer deprecation + a patch release instead.

12.3 Publish a Fix

# Create a fix changeset, version, and publish
pnpm changeset          # select patch for affected packages
pnpm version-packages
pnpm publish-packages
git push --follow-tags

13. Troubleshooting

Error: 402 Payment Required

Scoped packages default to private on npm. Ensure "access": "public" is set in .changeset/config.json (already done for GuideFlow) or pass --access public on the command line.

Error: 403 Forbidden — you must be a member of the org

Ask an organisation owner to add your npm username to the @guideflow org with developer (publish) permissions.

Error: ENEEDAUTH / need auth

Run npm login and verify with npm whoami. Check that ~/.npmrc has a valid token.

jsDelivr returns a 404 after publishing

jsDelivr indexes new npm package versions within a few minutes. Wait 5-10 minutes and try again. If the issue persists, verify the file path is correct by checking https://cdn.jsdelivr.net/npm/@guideflow/core@VERSION/ for the file listing.

jsDelivr returns stale / cached file

Purge the cache: curl -X POST "https://purge.jsdelivr.net/npm/@guideflow/core@VERSION/dist/index.js"

changeset publish skips packages

Changesets only publishes packages whose version in package.json is greater than what is on the registry. Make sure you ran pnpm version-packages first.

CI workflow does not publish

Double-check: (1) the NPM_TOKEN secret is set, (2) the token type is Automation, and (3) the workflow has NODE_AUTH_TOKEN in the environment.
