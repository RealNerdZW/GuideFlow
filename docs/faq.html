Frequently Asked Questions

General

What is GuideFlow.js?

GuideFlow.js is an open-source product tour library that uses a finite-state machine engine. It supports React, Vue, Svelte, and vanilla JavaScript, with optional AI-powered tour generation.

Does it have runtime dependencies?

No. @guideflow/core ships with zero runtime dependencies and is tree-shakeable.

What Node.js version is required?

Node.js ≥ 18.0.0 is required.

Installation

Do I need to install all packages?

No. Install only what you need. At minimum: @guideflow/core. Add a framework package (@guideflow/react, @guideflow/vue, or @guideflow/svelte) for framework-specific bindings.

Can I use GuideFlow.js via CDN?

Yes. The UMD build is available via jsDelivr: https://cdn.jsdelivr.net/npm/@guideflow/core@latest/dist/index.global.js

Usage

How do I create branching flows?

Use the on map in each FSM state to define transitions. For example, on: { NEXT: 'step-2', SKIP: 'step-3' } creates two possible paths from a state.

Can I persist tour progress across page reloads?

Yes. Pass persistence: 'localStorage' (or 'sessionStorage') to createGuideFlow(). You can also implement a custom ProgressStore.

Can I use GuideFlow.js with SSR (Next.js, Nuxt, SvelteKit)?

Yes. GuideFlow guards all DOM access with isBrowser checks. Start tours inside useEffect / onMounted / browser blocks to avoid hydration issues.

AI

Which AI providers are supported?

OpenAI (GPT-4o), Anthropic (Claude), Ollama (local), and a Mock provider for tests. You can also implement the AIProvider interface for any custom backend.

Is the AI package required?

No. @guideflow/ai is fully optional and only loaded if you call createAI().

Analytics

Which analytics platforms are supported?

PostHog, Mixpanel, Amplitude, Segment, and any HTTP webhook. You can also implement the AnalyticsTransport interface for custom destinations.

How does A/B testing work?

The ExperimentEngine uses a deterministic djb2 hash of userId + experimentId to assign variants client-side. The same user always gets the same variant without any server round-trip.
